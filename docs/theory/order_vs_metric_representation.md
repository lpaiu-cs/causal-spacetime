# Order Versus Metric Representation

Milestone 18 separates order structure, calibration structure, and metric
representation more explicitly.

## Order Layer

The order layer contains:

- causal trigger order among state-changing events,
- observer-chain order,
- radar-return order,
- observer-slice-relative distance order.

These structures compare events, ticks, returns, or pair distances. They do
not by themselves assign seconds, meters, ratios, velocities, metric tensors,
or curvature values.

## Calibration Layer

Calibration adds stronger constraints:

- clock processes,
- signal processes,
- matter standards,
- density or measure,
- repeated physical processes.

Positive monotone transformations preserve order but not ratios. Stable ratios
therefore require calibration, concatenation, repeated-process constraints, or
dynamics-like assumptions. A counting measure or density field is additional
structure, not a consequence of causal order alone.

## Metric Representation Layer

The metric representation layer contains:

- seconds,
- meters,
- ratios,
- metric tensors,
- curvature values,
- velocity,
- quantitative dynamics.

Metric geometry is not assumed fundamental here. It may be a stable calibrated
embedding of state-change order, observer-chain order, and observer-slice
distance order. The conformal ambiguity remains a key limitation: causal order
fixes light-cone or conformal structure in controlled models, while volume and
scale require measure or density input.

## Rejected Language

The following phrases are rejected formulations in this program:

- "distance = c times time slice",
- "causal order alone gives metric geometry",
- "spacetime is information-transfer speed",
- "time is information-transfer speed",
- "space is information-transfer speed".

The conservative claim is narrower: order structures can support effective
metric representations only after the relevant calibration, measure, observer,
and atlas-consistency assumptions are made explicit.
