# Deprecated Terminology: Observer-Chain Selection

This historical note is kept for readers of Milestone 20 outputs that still
contain older file names. The preferred terminology is now defined in
[reference_chain_selection.md](reference_chain_selection.md).

In the state-change trigger-network layer, arbitrary causal chains selected
from the order should be called reference chains. A reference-chain utility
score measures usefulness as a reference backbone for finite order-level
diagnostics. It is not an observer-reality score.

The term observer should be reserved for stronger protocol layers with
additional structure such as clock labels, radar rules, orientation references,
Rindler interpretation, or calibrated observer-protocol assumptions.
