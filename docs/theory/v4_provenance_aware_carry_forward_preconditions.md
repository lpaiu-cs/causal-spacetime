# V4 Provenance-Aware Carry-Forward Preconditions

Production v4 manifests must include measurement_protocol metadata,
profile_metadata, and handoff_provenance. Structured families must be
protocol_invariant, parameter-complete, admissible for production pairwise
response-profile dissimilarity, and have valid bottom-up, top-down, or hybrid
provenance.

Report-only controls must remain ineligible. Failed controls must remain
failed_control. A report-only or failed-control family can be diagnostic-complete
and still remain ineligible for production carry-forward.

These are admissibility preconditions, not performance evidence. They do not
replace the fixed M34 criteria and do not justify threshold retuning.

Milestone 45 applies fixed carry-forward criteria to the M44 v4 protocol bundle.
It does not regenerate manifests. It does not rerun fits. It does not retune
thresholds. It does not run stress tests. Carry-forward is stress-test
eligibility, not geometry.

## Milestone 46 Follow-Up

Milestone 46 audits why v4 protocol families remain blocked. It does not change
M45 decisions. It does not retune thresholds. It does not run stress tests. It
does not execute v5. V5 design is planned-only. Repeated remediation is an
overfitting risk unless justified by root-cause analysis.
