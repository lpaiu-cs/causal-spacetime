# Top-Down Handoff Manifests

A top-down handoff manifest is allowed only before evaluation. It freezes a
preregistered source of manifest structure, not a result. It must not use
held-out fit results, latent-fit outputs, carry-forward decisions, or stress-test
results to choose constraints.

A top-down manifest must record design source, design digest, frozen timestamp
or milestone, allowed data dependencies, forbidden data dependencies, and the
evaluation boundary. The manifest must still be tested by the same diagnostic
and carry-forward pipeline as other handoff manifests.

Top-down handoff is not evidence of geometry. Top-down handoff is allowed only
when preregistered and frozen before evaluation. The M41 top-down or hybrid
metadata changes manifest provenance accounting, not the fixed M34 criteria and
not any M38, M39, or M40 decision.

## Milestone 42 Boundary

Milestone 42 applies fixed carry-forward criteria to the M41 v3 protocol bundle.
Top-down/hybrid provenance is allowed only as preregistered manifest provenance,
not as evidence. M42 does not regenerate manifests, rerun fits, retune
thresholds, or run stress tests. Carry-forward is stress-test eligibility, not
geometry.

## Milestone 43 Follow-Up

Milestone 43 keeps top-down and hybrid provenance as admissibility metadata, not
as evidence. It audits why v3 protocol families remain blocked and
preregisters planned-only v4 remediation. It does not change M42 decisions. It
does not retune thresholds. It does not run stress tests. It does not execute
v4.
